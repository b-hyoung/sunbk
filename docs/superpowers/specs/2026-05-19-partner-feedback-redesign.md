# 협력사 피드백 반영 설계 — 용도 중심 재구성

날짜: 2026-05-19
출처: 2026-05-19 우빈이형 아버지 전화 피드백

## 배경

수연선박 실제 주력은 **통선·예항·측량업체 임대 (B2B)**인데, 현재 홈/필터가 "어선·화물선"만 노출 → 실제 고객(측량업체·건설사)이 사이트에서 자기 서비스를 못 찾음.

## 핵심 결정

### 1. 진입 동선: 선박 종류 → **용도** 중심

홈 메인 진입을 3개 용도 카드로 교체:

- 🛰️ **해상측량** → 측량업체 대여 (지오스토리·지오뷰·올포랜드 등 실적)
- 🚧 **해상공사** → 인원·물자 운반, 예항, 작업선
- 🚢 **화물·판매** → 화물 운송 및 매매

어선은 수연선박 임대 카테고리가 아닌 **shipyard(제일산업공사) 페이지 콘텐츠**로 (피드백: "어선은 조선소에 주로 옴").

### 2. 데이터 모델

`Vessel` 인터페이스에 새 필드 추가:

```ts
use_cases: ("survey" | "construction" | "cargo")[]
```

같은 배가 여러 용도에 등장 (예: 영진호 = `["survey", "construction"]`).

`vessel_type`(공식 분류)은 그대로 유지.

### 3. 서브카테고리

- 해상측량: **통선 / 도선 / FRP 작업선**
- 해상공사: **통선 / 예항·예인 / 작업선**
- 화물·판매: 없음

### 4. 홈 페이지 — 미니멀 (A안)

```
Hero (영상)
↓
🛰️ 해상측량  🚧 해상공사  🚢 화물·판매  (3장 카드)
↓
연락처 CTA
```

기존 섹션 제거: 통계(10척/인천/365일), 추천 선박 그리드.
`is_featured`는 데이터로 유지, /vessels 정렬용으로만 사용.

### 5. /vessels 선박 종류 필터 확장

현재: `어선 / 화물선` (2종) → **5종**:
`통선 / 예항선 / 작업선 / 도선 / 화물선`

vessel_type → UI 표시명 매핑:

| 데이터의 vessel_type | UI 표시 |
|---|---|
| `기타선(통선)(FRP)`, `기타선(통선)(강선)` | 통선 |
| `기선(강선)` (예항력 표기 있음) | 예항선 |
| `기선(FRP)`, `작업선` | 작업선 |
| `기타선(FRP)` | 도선 |
| `화물선` | 화물선 |

### 6. 가격 표기

가격이 null인 경우 "전화 협의" 표기, 임대·판매 분리 유지:

```
임대   전화 협의       (파란색)
판매   전화 협의       (회색)
```

VesselCard 뱃지 색 차별화:
- `type: "rent"` 또는 `"both"` → 파랑
- `type: "sale"` → 회색

/vessels 기본 정렬: 임대 가능 우선.
vessel 상세 페이지도 동일 정책.

## 구현 순서

1. `Vessel` 타입에 `use_cases` 추가 (`lib/supabase.ts`)
2. `constants/vessels-data.ts`의 각 배에 use_cases 채우기 (작업 이력 기반)
3. `vessel_type` → UI label 매핑 유틸 (`lib/vessel-types.ts` 신규)
4. 홈 페이지(`app/page.tsx`) 재작성 — Hero + 3 카드 + CTA만
5. `/vessels` 페이지: `use` 쿼리 파라미터 지원, 필터 5종으로 확장
6. `VesselFilter`: 종류 5개로 + use 카테고리 표시
7. `VesselCard`: 가격 null → "전화 협의", 뱃지 색 분리
8. `/vessels` 기본 정렬: 임대 가능 우선
9. vessel 상세(`app/vessels/[slug]/page.tsx`) 가격 표기 통일
