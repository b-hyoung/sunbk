# 제일산업공사(조선소) 통합 설계

날짜: 2026-04-16
대상 회사: 수연선박(임대·판매) + 제일산업공사(건조·수리) 2사 통합
*제3사(미정)는 이번 범위 밖 — 추후 확장 가능하도록 구조만 열어둠*

---

## 목적

같은 운영 주체가 사이트를 공유하되,
- 사용자는 **서비스 중심으로** 탐색하고
- 각 서비스의 **제공 주체(법인)는 작게 명시**해 B2B 거래 시 혼선 없도록 한다.

(B안 — 서비스 중심 + 회사 뱃지형)

---

## 공통 전제

- 두 회사는 **같은 주소·전화번호·이메일**을 사용 (`constants/company.ts`를 그대로 공유)
- 임시 이미지로 플레이스홀더 박스만 배치 (실제 조선소 사진은 이후 투입)
- 메인 브랜드 타이틀은 **변경 없음** (기존 "수연선박 | 선박 임대·판매 전문" 유지 — SEO·브랜드 인지 보호). 수리 페이지는 메타데이터만 별도 지정.

---

## 네비게이션 변경

**현재**
```
선박 목록 / 선박 임대 / 선박 판매 / 작업현장 / 회사소개 / 오시는길
```

**변경 후**
```
선박 목록 / 선박 임대·판매 / 선박 건조·수리 / 작업현장 / 회사소개 / 오시는길
```

구현: [constants/enums.ts](constants/enums.ts)의 `getNavLinks()` 수정.

- "선박 임대" + "선박 판매" → **"선박 임대·판매"** 한 항목으로 병합. 링크: `/vessels` (필터 없이 전체 목록)
- **"선박 건조·수리"** 신규 추가. 링크: `/shipyard`

## 페이지 구성 (`/shipyard`)

### 1. Hero
- 짧은 소개 1~2줄
- **서브타이틀 인라인 뱃지**: `선박 건조·수리 · 제일산업공사`
- 배경: 수수한 그레이 톤 혹은 플레이스홀더 이미지 (impeccable 원칙에 맞게 절제)
- 보조 CTA: 전화 버튼

### 2. 서비스 카테고리 (4개 카드)

| 카테고리 | 설명 |
|---|---|
| 신조 건조 | 신규 선박 설계·제작 |
| 선체·FRP 수리 | 파손부 복원, FRP 보강, 강판 교체 |
| 도장·방오 작업 | 선체 도장, 방오도료, 부식 방지 |
| 기관·설비 정비 | 엔진, 조타, 전기설비 점검·수리 |

- `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4`
- 아이콘은 lucide-react에서 적절히 선택 (Hammer, Wrench, PaintBucket, Settings 등)
- 현재 버전에선 카드 클릭 시 갤러리 해당 카테고리로 스크롤·필터링 (별도 상세 페이지 없음)

### 3. 갤러리 — Before/After 페어

- **페어 카드** 구조: 한 카드 내 좌측 "Before", 우측 "After" 이미지 2개 슬롯
- 현재 사진 없으니 **플레이스홀더 박스** (`bg-gray-100` + 아이콘 + "사진 준비 중" 텍스트)
- 페어 4~6개 배치 (카테고리 다양하게 예시)
- 카테고리 필터: 전체 / 신조 / 수리 / 도장 / 기관
- 각 페어에 작업 설명 1줄 + 카테고리 태그
- 모바일: 페어가 세로로 쌓이는 구조 (grid-cols-1), 데스크탑: 2열

### 4. 상담 CTA
- 전화 + 간단 문의 폼 (이름·연락처·내용만)
- 기존 `/booking/confirm` 재활용? → 아니, 별도 엔드포인트 불필요. 수연 booking API와 분리하려면 새 API 필요하지만, 현재는 **전화 CTA만**으로 시작하고 폼은 `tel:`과 기존 contact 페이지 링크로 대체.

## constants/enums.ts 변경안

```ts
export enum Routes {
  // ...기존...
  SHIPYARD = "shipyard",  // 신규
}

export function getNavLinks() {
  return [
    { href: `/${Routes.VESSELS}`,  label: "선박 목록" },
    { href: `/${Routes.VESSELS}`,  label: "선박 임대·판매" },  // type=rent/sale 제거
    { href: `/${Routes.SHIPYARD}`, label: "선박 건조·수리" },  // 신규
    { href: `/${Routes.WORK}`,     label: "작업현장" },
    { href: `/${Routes.ABOUT}`,    label: "회사소개" },
    { href: `/${Routes.CONTACT}`,  label: "오시는길" },
  ];
}
```

**⚠ 결정 필요**: "선박 목록"과 "선박 임대·판매"가 둘 다 `/vessels`로 가게 됨 (중복). 옵션:
- **(a)** "선박 목록" 제거 → 5개 메뉴 (`선박 임대·판매 / 선박 건조·수리 / 작업현장 / 회사소개 / 오시는길`)
- **(b)** 둘 다 유지하되 "선박 임대·판매"는 필터 없는 `/vessels`, "선박 목록"은 정렬·레이아웃 다른 별도 페이지
- **(c)** 반대로 "선박 임대·판매" 제거하고 "선박 목록"만 남김 (가장 보수적)

→ **사용자 리뷰 시 결정**. 기본 권장은 **(a)** — 중복 제거·직관성.
